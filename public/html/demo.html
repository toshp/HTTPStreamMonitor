Threshold Monitoring

A threshold monitor can be set to monitor a data stream to detect when a value exceeds or falls below a specific value.

Initialization

To initialize a min threshold monitor, you need to send a POST request with the following parameters in JSON to the endpoint.

POST http://jsondata.stream/min_threshold/init

{"client" : "cfcd208495", "key" : "AAPL", "min" : "142.00", "endpoint" : "https://hookb.in/vqL5kWdB"}
client
Your API key.
key
Maximum ten character "name" given to this monitor.
min
The min threshold value to monitor for. You will be notified if a value goes below this min value.
endpoint
The URL that will be notified when this threshold is crossed.

Initializing a max threshold monitor is similar, but you specify a maximum threshold instead. You will be notified if the data you send exceeds this max-value.

POST http://jsondata.stream/max_threshold/init

{"client" : "cfcd208495", "key" : "GE", "max" : "29.00", "endpoint" : "https://hookb.in/vqL5kWdB"}

Sending Data

To initialize a min threshold monitor, you need to send a POST request with the following parameters in JSON to the endpoint.

POST http://jsondata.stream/min_threshold/monitor

{"client" : "cfcd208495", "key" : "AAPL", "value" : "142.80", "timestamp" : "1493145062"}
client
Your API key.
key
The name of the monitor to match the value against.
value
The data value sent for monitoring.
timestamp
A time stamp (in any format) of when this data point was generated.
